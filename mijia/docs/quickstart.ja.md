# Mijia プラグイン · クイック操作ガイド (v1.1)

> **重要なヒント**：本プラグインは**自然言語による制御**を中心に設計されています。ほとんどの場合、デバイス ID や技術パラメータを調べる必要はなく、話し言葉のコマンドだけでデバイスを制御できます。

---

## 一、 コアコマンド早見表

AI Agent に対して以下の種類のコマンドを話しかけるだけで、プラグインが自動的に解析して実行します：

- **💡 デバイスのオン/オフ** — `寝室の照明をつけて` / `リビングのコンセントを切って`  
  「つけて／消して／オン／オフ」などの動詞に対応
- **🏠 エリア制御** — `リビングの照明をつけて` / `主寝室のエアコンを切って`  
  **(新版の目玉)** 「部屋名+デバイス名」に対応
- **🌡️ 温度調節** — `エアコンを 26 度にして` / `エアコンを 24 度に設定して`  
  数字と単位を自動認識
- **☀️ 明るさ調節** — `照明を 50% にして` / `デスクライトの明るさを 30 に`  
  パーセントまたは整数に対応
- **🔄 モード切替** — `エアコンを冷房にして` / `扇風機を自動に`  
  一般的なモード語に対応
- **🎬 シーン実行** — `帰宅モードを実行して` / `外出シーンを起動して`  
  あらかじめ Mijia App でシーンを設定しておく必要があります

## 二、 上級テクニック：「デバイスが多くて見つからない」問題の解決

家に同じ種類のデバイスが複数ある場合（例えば「照明」が複数ある）、`照明をつけて` とだけ言うと曖昧さ解消のプロンプトが表示されることがあります。

### 1. 曖昧さ解消プロンプトの例
コマンドが曖昧な場合、プラグインは次のようなプロンプトを返します。指示に従って**部屋名を追加**してください：

> 「照明」に一致するデバイスが 3 件見つかりました：
> 1. 🟢 リビング シーリングライト
> 2. 🔴 寝室 シーリングライト
> 部屋名+デバイス名で正確に指定してください。例：「寝室の照明」

### 2. 推奨される精確なコマンド形式
混乱を避けるため、**`[部屋名] + [デバイス名]`** の形を習慣にすることをおすすめします：

*   ❌ 曖昧なコマンド：`照明をつけて`
*   ✅ 精確なコマンド：`寝室の照明をつけて`
*   ✅ 精確なコマンド：`リビングのシーリングライトをつけて`

## 三、 よく使う API エントリ (開発者／上級ユーザー向け)

プラグインのインターフェースを直接呼び出したい場合は、以下が主要なエントリです：

- **`smart_control`** — 自然言語による統一制御 (推奨)  
  パラメータ：`{ "command": "寝室の照明をつけて" }`
- **`list_devices`** — デバイス一覧を取得  
  パラメータ：`{ "home_id": "12345" }`
- **`query_device_state`** — デバイス状態の取得  
  パラメータ：`{ "name": "エアコン" }`
- **`list_scenes`** — スマートシーン一覧を取得  
  パラメータ：`{ "home_id": "12345" }`

## 四、 よくある質問 (FAQ)

**Q: なぜ「未ログイン」と表示されるのですか？**
A: まず NEKO のプラグインパネルで Mijia プラグインを開き、「QR コードでログイン」をタップして認証を完了してください。

**Q: デバイスはオフラインなのに、プラグインではオンラインと表示されるのはなぜですか？**
A: プラグインに表示されるのはクラウド側の状態で、遅延が生じることがあります。デバイスが実際に通電して Mijia App に接続されているか確認してください。

**Q: どのデバイスに対応していますか？**
A: 理論上、Mijia IoT プラットフォーム上のすべてのデバイスに対応しています。Mijia App で制御できるデバイスであれば、本プラグインでも制御可能です。
