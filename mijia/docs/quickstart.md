# Mijia 插件 · 快速操作手册 (v1.1)

> **核心提示**：本插件主打**自然语言控制**。绝大多数情况下，您不需要查找设备ID或技术参数，直接用口语指令即可控制设备。

---

## 一、 核心指令速查表

只需对 AI Agent 说出以下类型的指令，插件会自动解析并执行：

- **💡 开关设备** — `打开卧室灯` / `关掉客厅插座`  
  支持"打开/关闭/开/关"等动词
- **🏠 区域控制** — `打开客厅的灯` / `关闭主卧空调`  
  **(新版重点)** 支持"房间名+设备名"
- **🌡️ 调节温度** — `空调调到26度` / `空调设置24度`  
  自动识别数字和单位
- **☀️ 调节亮度** — `灯调到50%` / `台灯亮度30`  
  支持百分比或整数
- **🔄 切换模式** — `空调调制冷` / `风扇调自动`  
  支持中文模式词
- **🎬 执行场景** — `执行回家模式` / `触发离家场景`  
  需在米家 App 中预先设置好场景

---

## 二、 进阶技巧：解决"设备太多找不到"的问题

如果您家里有多个同类型设备（例如多个"灯"），直接使用 `打开灯` 可能会触发消歧提示。

### 1. 消歧提示示例
当指令不明确时，插件会返回类似以下的提示，请根据提示**再次补充房间名**：

> 找到 3 个匹配 '灯' 的设备：
> 1. 🟢 客厅 吸顶灯
> 2. 🔴 卧室 吸顶灯
> 请用房间名+设备名精确指定，如 '卧室灯'

### 2. 推荐的精确指令格式
为了避免混淆，建议养成使用 **`[房间名] + [设备名]`** 的习惯：

*   ❌ 模糊指令：`打开灯`
*   ✅ 精确指令：`打开卧室灯`
*   ✅ 精确指令：`打开客厅的吸顶灯`

## 三、 常用 API 入口 (供开发者/高级用户)

如果您需要直接调用插件接口，以下是核心入口说明：

- **`smart_control`** — 自然语言统一控制 (推荐)  
  参数：`{ "command": "打开卧室灯" }`
- **`list_devices`** — 获取设备列表  
  参数：`{ "home_id": "12345" }`
- **`query_device_state`** — 查询设备状态  
  参数：`{ "name": "空调" }`
- **`list_scenes`** — 获取智能场景列表  
  参数：`{ "home_id": "12345" }`

## 四、 常见问题 (FAQ)

**Q: 为什么提示"未登录"？**  
A: 请先在 Neko 插件面板中找到 Mijia 插件，点击"扫码登录"并完成授权。

**Q: 为什么设备离线但插件显示在线？**  
A: 插件显示的是云端状态，可能存在延迟。请检查设备是否真的通电并连接到米家 App。

**Q: 支持哪些设备？**  
A: 理论上支持所有米家 IoT 平台设备。只要能在米家 App 中控制的设备，本插件均可支持。
