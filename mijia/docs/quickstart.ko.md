# Mijia 플러그인 · 빠른 사용 매뉴얼 (v1.1)

> **핵심 안내**: 본 플러그인은 **자연어 제어**를 중심으로 설계되었습니다. 대부분의 경우 장치 ID나 기술 파라미터를 찾을 필요 없이, 일상적인 말투의 명령으로 장치를 제어할 수 있습니다.

---

## 1. 핵심 명령어 빠른 표

AI Agent 에게 아래 유형의 명령을 말하기만 하면, 플러그인이 자동으로 해석하고 실행합니다:

- **💡 장치 켜기/끄기** — `침실 조명 켜줘` / `거실 콘센트 꺼줘`  
  "켜줘/꺼줘/켜/꺼" 등의 동사 지원
- **🏠 구역 제어** — `거실 조명 켜줘` / `안방 에어컨 꺼줘`  
  **(새 버전 핵심)** "방 이름 + 장치 이름" 지원
- **🌡️ 온도 조절** — `에어컨 26도로 맞춰` / `에어컨 24도로 설정해`  
  숫자와 단위를 자동 인식
- **☀️ 밝기 조절** — `조명 50%로 맞춰` / `스탠드 밝기 30`  
  퍼센트 또는 정수 지원
- **🔄 모드 전환** — `에어컨 냉방으로 바꿔` / `선풍기 자동으로 바꿔`  
  일반적인 모드 단어 지원
- **🎬 씬 실행** — `귀가 모드 실행해` / `외출 씬 작동해`  
  Mijia App 에서 씬을 미리 설정해 두어야 합니다

## 2. 고급 팁: "장치가 너무 많아 찾기 어려운" 문제 해결

집에 같은 유형의 장치가 여러 개 있는 경우(예: 여러 개의 "조명"), `조명 켜줘`라고만 말하면 모호성 해소 안내가 표시될 수 있습니다.

### 1. 모호성 해소 안내 예시
명령이 모호할 때, 플러그인은 아래와 같은 안내를 반환합니다. 안내에 따라 **방 이름을 다시 보충**해 주세요:

> '조명'에 해당하는 장치 3개를 찾았습니다:
> 1. 🟢 거실 실링라이트
> 2. 🔴 침실 실링라이트
> 방 이름 + 장치 이름으로 정확하게 지정해 주세요. 예: '침실 조명'

### 2. 권장하는 정확한 명령 형식
혼동을 피하기 위해 **`[방 이름] + [장치 이름]`** 형식을 습관처럼 사용하시기를 권장합니다:

*   ❌ 모호한 명령: `조명 켜줘`
*   ✅ 정확한 명령: `침실 조명 켜줘`
*   ✅ 정확한 명령: `거실 실링라이트 켜줘`

## 3. 자주 사용하는 API 엔트리 (개발자/고급 사용자용)

플러그인 인터페이스를 직접 호출해야 한다면, 핵심 엔트리는 다음과 같습니다:

- **`smart_control`** — 자연어 통합 제어 (권장)  
  매개변수: `{ "command": "침실 조명 켜줘" }`
- **`list_devices`** — 기기 목록 가져오기  
  매개변수: `{ "home_id": "12345" }`
- **`query_device_state`** — 장치 상태 조회  
  매개변수: `{ "name": "에어컨" }`
- **`list_scenes`** — 스마트 시나리오 목록 가져오기  
  매개변수: `{ "home_id": "12345" }`

## 4. 자주 묻는 질문 (FAQ)

**Q: 왜 "로그인되지 않음"이라고 표시되나요?**
A: 먼저 NEKO 플러그인 패널에서 Mijia 플러그인을 찾아 "QR 코드 로그인"을 누르고 인증을 완료해 주세요.

**Q: 장치는 오프라인인데 플러그인에서 온라인으로 표시되는 이유는 무엇인가요?**
A: 플러그인이 표시하는 것은 클라우드 상태이며, 지연이 있을 수 있습니다. 장치가 실제로 전원이 들어와 있고 Mijia App 에 연결되어 있는지 확인해 주세요.

**Q: 어떤 장치를 지원하나요?**
A: 이론상 모든 Mijia IoT 플랫폼 장치를 지원합니다. Mijia App 에서 제어할 수 있는 장치라면 본 플러그인에서도 제어할 수 있습니다.
